# Google Drive Markdown Preview

Google Drive のプレビュー画面で markdown ファイルをレンダリングする Chrome 拡張です。
生テキストの表示を GitHub 風に整形された HTML に置き換えます。

[English README is here](README.md)

## 特徴

- **自動検出**：`.md` / `.markdown` ファイルをプレビューで開くと自動的にレンダリングします
- **表示切り替え**：「Show Raw」ボタンでレンダリング結果と元のテキストを切り替えられます
- **GitHub 風スタイル**：読みやすい整形に加え、popup からダークテーマを選択できます
- **ファイル移動への追従**：プレビュー内でキーボード操作により別ファイルへ移動しても再レンダリングされます
- **ローカル処理のみ**：パースとサニタイズはすべてブラウザ内で完結し、拡張はネットワーク通信を行いません

## インストール

この拡張は Chrome Web Store に公開していません。
手動でインストールします。

1. このリポジトリをダウンロードまたはクローンする
2. `chrome://extensions/`（Chromium 系ブラウザの拡張機能ページ）を開く
3. 「デベロッパーモード」を有効にする
4. 「パッケージ化されていない拡張機能を読み込む」でリポジトリのディレクトリを選択する

注意：Chrome Web Store には同名の拡張が存在しますが、このリポジトリとは無関係です。
インストール済みの場合は、二重レンダリングを避けるため無効化してください。

## 使い方

1. Google Drive を開く
2. `.md` ファイルをダブルクリックしてプレビューを開く
3. 生テキストが自動的にレンダリング結果へ置き換わる
4. 表示を切り替えたいときは、コンテンツ上部の「Show Raw」ボタンを使う
5. 設定はツールバーの拡張アイコンから開く

## 対応する markdown 記法

パースには [marked](https://github.com/markedjs/marked) v4.3.0 を GFM 有効で使用しています。

- 見出し（H1–H6）
- テキスト装飾（太字、斜体、打ち消し線）
- 順序付きリストと順序なしリスト（ネスト可）
- リンクと画像
- インラインコードとコードブロック（スタイルは当たりますが、シンタックスハイライトはありません）
- テーブル
- 引用ブロック
- 水平線

## 設定

ツールバーの拡張アイコンをクリックすると設定を開けます。

- **有効/無効**：拡張のオン・オフを切り替えます（無効化すると元の生テキスト表示に戻ります）
- **テーマ**：ライトまたはダークを選択します
- **ステータス**：Google Drive のページを開いているかどうかを表示します

## 動作の仕組み

- MutationObserver で Drive のプレビュー要素（`[role="document"]`）の追加を監視します
- ファイル名はローカライズされた `aria-label`（英語：`Displaying foo.md`、日本語：`「foo.md」を表示しています`）から抽出するため、UI 言語を問わず検出できます
- Drive のクラス名は難読化されており時期によって変わるため、プレビュー本体は構造（プレビュー要素内の最大の `<pre>`）で特定します
- 生成した HTML は挿入前に [DOMPurify](https://github.com/cure53/DOMPurify) v3.2.6 でサニタイズします
- content script は全フレームで実行される（`all_frames: true`）ため、Drive がプレビューを同一オリジンの iframe 内に置いた場合でも動作します

開発環境の構築とアーキテクチャの詳細は [CLAUDE.md](CLAUDE.md) を参照してください。

## 権限とプライバシー

- ホストへのアクセスは `https://drive.google.com/*` に限定しています
- `storage` は設定 2 項目（有効/無効とテーマ）の保存のみに、`activeTab` は popup でのページ状態の表示のみに使います
- データの収集や外部送信は行いません。marked と DOMPurify は `lib/` 配下にローカル同梱しています

## 対応ブラウザ

Manifest V3 に対応した Chromium 系ブラウザ（Chrome 88 以降、Edge、Brave、Vivaldi など）で動作します。
開発と動作確認は Vivaldi で行っています。

## トラブルシューティング

**レンダリングされないとき**

- popup と `chrome://extensions/` の両方で拡張が有効になっているか確認する
- Google Drive のタブをリロードする
- Chrome Web Store からインストールした同名拡張があれば無効化する

**デバッグ**

- `content.js` 先頭の `GDMD_DEBUG` を `true` にして拡張をリロードすると、Drive タブの DevTools コンソールに `[GDMD]` ログが出力されます

## 変更履歴

### Version 1.2.0

- ロケール非依存の検出（英語の「Displaying」への一致ではなく、aria-label からのファイル名抽出に変更し、日本語 UI などでも動作）
- MutationObserver によるイベント駆動の検出（ポーリングと固定タイムアウトを廃止）
- 難読化クラス名ではなく構造（プレビュー要素内の最大の `<pre>`）によるコンテンツ特定
- DOMPurify による HTML のサニタイズ
- content script の全フレーム実行（`all_frames: true`）
- popup のテーマ設定の反映（`.gdmd-dark` クラスによるダークテーマ）

### Version 1.1.0

- 実運用向けリリース
- パフォーマンスとポーリング機構の最適化
- 複数ファイル対応の強化
- 信頼性とエラーハンドリングの改善

### Version 1.0.0

- 初回リリース
- markdown の検出とレンダリング
- 生テキストとレンダリング結果の切り替え
- 設定 popup
